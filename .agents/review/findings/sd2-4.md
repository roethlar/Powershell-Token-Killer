# sd2-4: Foreground non-execution guard breaks on apostrophes in the temp path

**Severity**: LOW — host-path-specific battery failure (e.g. a Windows
profile like C:\Users\O'Brien); the wiring itself is correct.
**Status**: Open
**Branch**: master (direct, per recorded precedent)
**Commit**: (filled in after commit)

## Evidence
`server/PtkMcpServer.Tests/ShellDialectWiringTests.cs`
(Foreground_bash_only_script_is_refused_and_nothing_executes) interpolates
the probe path into a single-quoted PowerShell literal unescaped:
`$"export X=1; New-Item -ItemType File -Path '{probe}'"`.

## Predicted observable failure
An apostrophe in the temp path terminates the literal early → the script
carries a TerminatorExpected parse error → the detector's parse-fatal
branch matches no frozen error id → no refusal → the script "runs" into an
ordinary parse error and the test's refusal assertions fail on a correct
implementation.

## What
Test-code portability defect: PowerShell-literal interpolation without
PowerShell-literal escaping.

## Approach
Escape the interpolated path (`probe.Replace("'", "''")`) and make the
guard self-proving on every host by planting a literal apostrophe in the
probe file name itself — the escaping is then exercised unconditionally,
not only on exotically-named machines.
