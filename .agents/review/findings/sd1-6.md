# sd1-6: space-filler blanking can SYNTHESIZE a bash shape (new false positive)

**Severity**: MEDIUM — the ceae832 blanking introduced an over-match class,
directly disproving sd1-3's "never an over-match" known-gap claim; violates
the frozen precision-first rule.
**Status**: RESOLVED — closed at re-grade round 2 (head `293eda6`)
**Branch**: master (direct), per loop precedent

## Evidence
Reported by codex at re-grade round 1 (head `acb0f39`): blanking replaces
string extents with spaces, and the shape regexes then match ACROSS the
blank where the original text would not have matched.
**Master verified live at head:** `foo 'bar'() { echo hi; }` — the raw text
does not match the function-shape regex, but blanking `'bar'` yields
`foo      () { echo hi; }`, which does; the detector returns
`a bash function definition (name() { ... })`.

## Predicted observable failure
An unparsable script whose only "bash function shape" is fabricated by the
space filler gets bash-function guidance for an unrelated pwsh parse error —
wrong dialect guidance (sd1-3 harm class), but INTRODUCED by the fix rather
than pre-existing.

## Approach (proposed, fix round 2)
Blank with a filler that cannot extend regex matches across the erased span
(e.g. a non-space sentinel char excluded by the shape regexes' character
classes), or mark blanked spans and reject shape matches that overlap one.
Guard: the exact repro expects null; existing IN tests keep recall.
