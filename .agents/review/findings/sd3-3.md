# sd3-3: Job-poll recovery note fires on marker text the job itself printed

**Severity**: LOW — a spurious note prompts one unnecessary log retrieval;
no wrong execution.
**Status**: In progress
**Branch**: master (direct, per recorded precedent)
**Commit**: (filled in after commit)

## Evidence
`server/PtkMcpServer/Tools/JobTool.cs` (sd3-2 fix) infers elision from an
origin-blind substring search of the shaped poll, while the module
preserves under-limit passthrough text unchanged — so a job whose OWN
output contains the marker wording (a background grep over this repo's
source, a job cat-ing previously elided output) receives the recovery
note despite complete output.

## Predicted observable failure
An under-limit poll carrying job-printed marker text appends "raw=true
does not apply here - the complete raw log is <path>", prompting a
pointless read of a log that contains nothing beyond what the poll
already showed.

## What
The note must key on "shaping elided THIS poll", not "the poll contains
elision-marker-shaped text".

## Approach
Round 1 (302891a) hardened the heuristic: anchored whole-line regex +
"shaping shortened the text" gate. **Re-grade round 3 broke it in both
directions** (both probes master-verified before triage): ANSI stripping
shortens under-limit text without eliding (81→72, false note), and
near-boundary elision LENGTHENS text (401 one-char lines, 801→872 — the
genuine note suppressed; filed as sd3-4). Round 2 (final) abandons
downstream inference entirely: `Limit-PtcPassthrough`/`Compress-PtcOutput`
accept an `-ElisionHint`, `ShapeTextAsync` forwards it, and JobTool passes
"read the complete raw log at <path> if the elided middle matters" — the
marker's advice is composed BY the elision itself and can be neither
false nor missing. The former rtk-log-leg residual disappears (the hint
rides every text leg).

## Files changed
- `src/PwshTokenCompressor.psm1` — `-ElisionHint` on Limit-PtcPassthrough
  and Compress-PtcOutput (all three Limit call sites).
- `server/PtkMcpServer/RunspaceHost.cs` — ShapeTextAsync forwards the hint.
- `server/PtkMcpServer/Tools/JobTool.cs` — passes the job-context hint;
  detection heuristic deleted.
- `server/PtkMcpServer.Tests/RawUsageTests.cs` — false-positive guard now
  covers plain AND ANSI-colored marker copies ("raw log" absence probe);
  genuine-elision guard asserts the in-marker hint.
- `tests/PwshTokenCompressor.Tests.ps1` — new hint-composition test.
